Sometimes youâ€™ll need to make checks in your statechartâ€™s current state _without_ receiving an event. You can do this with an eventless transition.

Eventless transitions are transitions without events. These transitions are **always** taken after any transition in their state if enabled. Eventless transitions are labeled â€œalwaysâ€ and often referred to as â€œalwaysâ€ transitions.

A simple example is a state that always transitions from `a` to `b`:

```ts
import { createMachine } from 'xstate';

const machine = createMachine({
  initial: 'a',
  states: {
    a: {
      always: [
        {
          target: 'b',
        },
      ],
    },
    b: {},
  },
});
```

Using the `always` transition means that `a` will _instantly_ transition to `b` when the machine enters the `a` state.

## Adding guards

You can also pair `always` transitions with a guard:

```ts
import { createMachine } from 'xstate';

const machine = createMachine(
  {
    initial: 'a',
    states: {
      a: {
        // highlight-start
        always: [
          {
            cond: 'shouldTransition',
            target: 'b',
          },
        ],
        // highlight-end
      },
      b: {},
    },
  },
  {
    guards: {
      shouldTransition: (ctx) => ctx.user.role === 'admin',
    },
  },
);
```

In the example above, the transition will only happen when `shouldTransition` returns true. Otherwise, the machine will stay in the same state.

â€œAlwaysâ€ transitions are checked immediately when the machine enters the state node, after checking for regular transitions and before checking if there are any transitions for any other queued events.

## Reducing duplication

â€œAlwaysâ€ transitions are extremely useful for reducing duplication in guards, along with other uses.

Example without `always`:

```ts

import { createMachine, assign } from 'xstate';

const gameMachine = createMachine(
  {
    initial: 'playing',
    context: {
      points: 0,
    },
    states: {
      // highlight-start
      playing: {
        on: {
          AWARD_POINTS: [
          // highlight-end
            {
              cond: 'didPlayerWin',
              actions: 'awardPoints',
              target: 'win',
            },
            {
              cond: 'didPlayerLose',
              actions: 'awardPoints',
              target: 'lose',
            },
            {
              actions: 'awardPoints',
            },
          ],
          MAKE_MOVE: [
            {
              cond: 'didPlayerWin',
              actions: 'doMove',
              target: 'win',
            },
            {
              cond: 'didPlayerLose',
              actions: 'doMove'
              target: 'lose',
            },
            {
              actions: 'doMove'
            },
          ],
        },
      },
      win: {},
      lose: {},
    },
  },
  {
    actions: {
      awardPoints: assign({
        points: (context) => context.points + 100,
      }),
      makeMove: assign({
        points: (context) => context.points + Math.floor(Math.random() * 10),
      }),
    },
    guards: {
      didPlayerWin: (context, event) => {
        return context.points > 99;
      },
      didPlayerLose: (context, event) => {
        return context.points < 0;
      },
    },
  }
);
```
