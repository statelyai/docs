When you run a statechart, it becomes an *actor*, a running process that can receive events. Often, youâ€™ll need your actor to run _other actors_; spawning new statecharts, waiting for promises, or subscribing to observables. We call these child actors *services*; actors whose state is managed by another actor.

We use the `invoke` attribute on a state to _invoke_ a service in our machine. You can invoke a service on any state, including the root node.

```ts twoslash
import { createMachine } from "xstate";

const machine = createMachine(
  {
    invoke: {
      src: "myService",
    },
  },
  {
    services: {
      /**
       * The service is defined here
       */
      myService: async () => {},
    },
  },
);
```

You can also run several invocations at the same time by specifying `invoke` as an array:

```ts twoslash
import { createMachine } from "xstate";

const machine = createMachine(
  {
    invoke: [
      {
        src: "myService",
      },
      {
        src: "myOtherService",
      },
    ],
  },
  {
    services: {
      myService: async () => {},
      myOtherService: async () => {},
    },
  },
);
```

Running several invocations simultaneously is useful when you want several sub-processes running on the same state.
